# 连锁口腔机构客服中心回访任务服务

面向连锁口腔机构的后端回访任务服务系统，专门承接各门店术后患者提醒和分派规则管理。

## 功能模块

### 1. 规则配置模块
- 总部运营人员配置不同治疗类型的回访节奏
  - 拔智齿当天晚间问出血
  - 根管治疗后两天问疼痛
  - 正畸拔牙后七天提醒复查
- 配置标准问询话术模板
- 配置异常关键词（如"持续出血"、"发热"、"麻木"），命中后自动转医生复核

### 2. 坐席任务模块
- 治疗记录进入任务池后，按门店、医生、患者风险标签智能分派
- 坐席工作台显示：待拨打名单、标准话术、处理按钮
- 处理结果：接通、未接、拒接、关机、空号等
- 关键词命中或手动触发 → 转为医生复核任务
- 支持任务转派、超时自动标记

### 3. 统计看板模块
- 全系统总览统计（完成率、异常率、超时等）
- 各门店维度统计对比
- 各坐席/护士/医生维度统计
- 超时未回访任务清单

## 技术栈

- **后端框架**: FastAPI 0.104.1
- **数据库**: SQLite (默认) / 支持 PostgreSQL / MySQL
- **ORM**: SQLAlchemy 2.0.23
- **数据验证**: Pydantic 2.5
- **认证**: JWT (python-jose)
- **密码**: bcrypt (passlib)

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 初始化数据库和示例数据

```bash
python scripts/init_data.py
```

### 3. 启动服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 4. 访问接口文档

- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## 核心业务流程

```
治疗记录录入
      ↓
按治疗类型匹配激活的回访规则
      ↓
生成回访任务（按规则天数+时段排期）
      ↓
任务池自动分派（考虑：门店归属、风险标签、坐席负载）
      ↓
坐席领取/开始任务 → 查看标准话术 → 拨打
      ↓
记录处理结果
   ├── 正常接通 → 任务完成
   ├── 未接通/异常 → 标记异常
   └── 关键词命中 / 手动 → 转医生复核
            ↓
         医生复核 → 填写意见 → 闭环
```

## 任务分派逻辑

分派时综合考虑：
1. **门店匹配**：门店护士优先分派本店任务
2. **风险分级**：高风险患者优先分派给门店护士
3. **负载均衡**：按坐席当前待办数 - 今日完成数打分
4. **角色偏好**：低风险任务偏好总部客服坐席
